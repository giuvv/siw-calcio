Analisi Prestazioni — Fetch Strategy

Confronto tra Fetch LAZY e JOIN FETCH per il caso d'uso: visualizzazione dettaglio squadra con giocatori.

Il problema N+1

Quando si carica una Squadra e si accede poi ai suoi Giocatori, JPA/Hibernate deve fare una scelta: caricarli subito (EAGER) o aspettare che vengano richiesti (LAZY).

Con la strategia LAZY, il codice squadra.getGiocatori().size() scatena una seconda query SQL separata. Per 1 squadra questo è 1+1 = 2 query. Se si caricassero N squadre e per ognuna si accedessero i giocatori, il totale salirebbe a N+1 query.

Con JOIN FETCH (query JPQL personalizzata nel repository), Hibernate recupera squadra e giocatori in una sola query con LEFT JOIN, eliminando completamente il problema.

Le due strategie nel codice

❌ Strategia 1 — LAZY (N+1)

// SquadraService.java
Squadra s = squadraRepository
    .findById(id)        // Query 1: SELECT squadra
    .orElse(null);

s.getGiocatori().size(); // Query 2: SELECT giocatori
                         // (scatenata accedendo alla lista)

SQL generato da Hibernate:
SELECT * FROM squadra WHERE id = ?
SELECT * FROM giocatore WHERE squadra_id = ?

✅ Strategia 2 — JOIN FETCH (1 query)

// SquadraRepository.java
@Query("SELECT s FROM Squadra s " +
       "LEFT JOIN FETCH s.giocatori " +
       "WHERE s.id = :id")
Squadra findByIdWithGiocatoriEager(
    @Param("id") Long id);

SQL generato da Hibernate:
SELECT s.*, g.* FROM squadra s
LEFT JOIN giocatore g
ON g.squadra_id = s.id
WHERE s.id = ?

Nota sui tempi: Con pochi dati di test, la differenza in millisecondi è piccola perché PostgreSQL tiene i dati nella propria cache di buffer dopo la prima query. In uno scenario reale con centinaia di squadre e migliaia di giocatori, il vantaggio del JOIN FETCH diventerebbe molto più evidente. L'aspetto più importante è il numero di query: 2 vs 1. Con N squadre, il problema LAZY degenerebbe in N+1 query totali.

Le query SQL eseguite sono visibili nel log della console Spring Boot grazie a spring.jpa.show-sql=true già attivo in application.properties.
